# 目标

实现一个今日新闻资讯收集的智能体(agent), 用来收集、整理、聚合用户指定类型的新闻资讯, 由agent自主搜集信息并整合为一份资讯报告;

# 可能用到的技术

opencode sdk, claude code sdk, mcp技术, 上下文工程, 提示词, skills, subagent, 多智能体协作

# 目标评级

## C: 构建基础的agent, 能正确的拉取今日某类型的新闻,并生成报告

- 用户通过 自然语言交互的方式，说明需要获取的新闻类型,例如：帮我收集体育运动类（国际政治类、娱乐新闻）资讯；
- agent根据用户要求，从互联网上获取指定类型的信息，从信息获取到报告产出过程无人工干预
- 生成报告应是一个文件夹

```
文件结构示例
- report(文件夹)
---- index.md (目录)
---- 新闻1.md
---- 新闻2.md
...
```

- 新闻必须是今日的新闻内容
- 具体的新闻文件应由摘要和内容两部分组成
- 新闻文件最后应该包括新闻来源，例如引用的网址url等;

### 可能需要学习的技术

框架: opencode sdk, claude code skd
能自行编写mcp
提示词编写

## B: 在C的基础上

- 支持用户通过 自然语言同时指定多个新闻类型，例如：帮我收集体育运动和国际政治相关新闻；
- 图文并茂
- 对收集到的今日相关资源分类做汇总和摘要；
- 在处理更加细化的分支领域内仍能表现良好, 且运行时间在可控范围内
- 生成报告应是一个文件夹

```
文件结构示例
- report/
-- 资讯分类1/
--- 资讯汇总与摘要/
---- index.md (目录)
---- 新闻1.md
---- 新闻1/
----- 图片1.png
----- 图片2.png
---- 新闻2.md
...
```

### 可能需要学习的技术

skills, subagent

## A: 在B的基础上

- 支持更模糊的 自然语言指定新闻类型，例如：我最近在学习人工智能相关的知识，请帮我收集相关资讯；
- 支持按日期、资讯分类输出结构；
- 资讯汇总与摘要中内容 需要可以链接到新闻详情文件。即用户可以直接在摘要文件中跳转查看新闻详细内容；
- 能保证新闻的真实性和可追溯性, 且有针对真实性和可追溯性的论据
文件结构示例
- report/
-- 资讯分类1/
--- 日期/
---- 资讯汇总与摘要/
----- index.md (目录)
----- 新闻1.md
----- 新闻1/
------ 图片1.png
------ 图片2.png
----- 新闻2.md
...

```

### 可能需要学习的技术:

## S: 在A的基础上
- 针对资讯关键点，构建该关键内容的发展时间轴，展示与之相关的各项新闻资讯。例如国际政治下 美国大选，展示近期美国大选相关所有内容；
- 针对资讯关键词，给出符合逻辑的事件发展推理与预测，例如国际金融下 国际金价 关键词，可以展示国际金价变化的相关新闻报告，同时给出国际金价变化趋势的预测以及预测理由；
- 针对各条新闻,能展示该新闻从事件最开始到今日的整个时间线发展脉络;
