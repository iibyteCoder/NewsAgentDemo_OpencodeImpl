# A级目录结构模板

本文件定义了A级目标的报告目录结构。

## 目录结构

```
report_YYYYMMDD_HHMMSS/
├── 分类1/
│   ├── YYYY-MM-DD/
│   │   ├── 资讯汇总与摘要/          ← 文件夹（包含该日期的所有事件）
│   │   │   ├── index.md            ← 该日期的目录索引
│   │   │   ├── 事件标题1.md        ← 事件1的完整报告
│   │   │   │   ├── 事件标题和摘要
│   │   │   │   ├── 来源列表（所有相关URL）
│   │   │   │   ├── 真实性验证
│   │   │   │   ├── 事件时间轴
│   │   │   │   └── 趋势预测
│   │   │   ├── 事件标题2.md        ← 事件2的完整报告
│   │   │   └── 事件标题3.md        ← 事件3的完整报告
│   └── index.md                    ← 该分类的总索引
├── 分类2/
│   └── ...
└── index.md                        ← 总索引
```

## 核心概念说明

### 事件 (Event)

一个"事件"是指**多个媒体报道同一件事**的聚合。例如：
- "特朗普赢得艾奥瓦初选" 可能被 CNN、Fox、Reuters 等多个媒体报道
- 这些报道聚合为一个"事件"，包含：
  - 统一的标题和摘要
  - 所有来源 URL 列表
  - 真实性验证结果
  - 事件时间轴
  - 趋势预测

### 处理流程

1. **收集**: 搜索并获取新闻内容
2. **聚合**: 将多个 URL 聚合为事件
3. **验证**: 验证每个事件的真实性
4. **时间轴**: 构建每个事件的发展脉络
5. **预测**: 预测每个事件的未来发展
6. **报告**: 生成结构化报告
