# 事件摘要模板

此模板定义了事件摘要部分的格式。

## 模板格式

{event_summary}

## 说明

### 摘要要求

1. **长度**：150-200字
2. **结构**：2-3段或连贯的一段
3. **要素**：
   - 事件是什么
   - 何时发生
   - 涉及谁
   - 为何重要

### 内容组织

**第1段：事件概述**（必须）

- 简洁描述事件核心内容
- 包含时间和关键事实
- 1-2句话

**第2段：核心信息**（可选，如果信息丰富）

- 关键数据
- 主要发展
- 重要细节

**第3段：重要意义/影响**（可选，如果重要）

- 为何重要
- 有什么影响
- 意味着什么

### 占位符说明

- `{event_summary}` - 完整的事件摘要内容

### 示例

**金融类事件**：

```markdown
2026年1月30日，国际金融市场见证了历史上最剧烈的金价波动。在连续三日突破六道重要关口、触及5598.75美元/盎司历史新高后，金价于1月29日深夜突然"史诗级跳水"，30分钟内暴跌493美元至5100美元附近，单日最大跌幅达8.8%，刷新历史波动记录，导致全球黄金市值蒸发约3.5万亿美元。这场史无前例的剧烈震荡引发了全球投资者的高度关注，标志着国际黄金市场进入了前所未有的不确定性时期。
```

**科技类事件**：

```markdown
2026年1月30日，OpenAI正式发布GPT-5模型，在多项基准测试中超越人类专家水平。新模型在推理能力、多模态理解和代码生成方面实现突破性进展，支持100万token的上下文窗口。发布后24小时内，已有超过500家企业申请API访问权限，标志着人工智能技术进入新的发展阶段。
```

### 注意事项

- ⭐⭐⭐ **简洁精炼** - 控制在150-200字以内
- ⭐⭐⭐ **5W1H完整** - 包含What、When、Who、Where、Why、How
- ⭐⭐ **信息密集** - 每句话都包含关键信息
- ⭐ **客观准确** - 基于新闻事实，不添加主观判断
- ⭐ **流畅易读** - 句子之间逻辑连贯
